# Product Roadmap

## ✅ Completed (v1.0)

- ✅ Core execution engine (retention, erasure, masking)
- ✅ Policy DSL with Zod validation
- ✅ Offline license verification (Ed25519)
- ✅ Soft run limits (`max_runs_per_day`)
- ✅ Hash-chained tamper-evident audit logs
- ✅ Docker images with multi-platform support
- ✅ CLI with dry-run mode
- ✅ Comprehensive test coverage (98.85%)

## 🔄 In Progress

- SBOM generation
- Docker image signing
- Helm charts for Kubernetes

## 📋 Planned (Q1 2026)

- MySQL/MariaDB support
- MongoDB support
- Policy versioning and rollback
- Audit log encryption at rest
- Webhook notifications for audit events

## 🎯 Strategic Direction

**What Retentix Will Always Be:**
- Headless execution engine
- Self-hosted / on-premises
- CLI-first interface
- Deterministic and auditable
- Zero network dependencies

**What Retentix Will Never Be:**
- A SaaS platform
- A compliance dashboard
- A data discovery tool
- A multi-tenant service
- A UI-first product

## 💡 Philosophy

Retentix focuses on **execution**, not visibility.

We believe:
1. Compliance actions must be deterministic
2. Audit trails must be tamper-evident
3. Execution must be air-gap compatible
4. Simplicity enables trust

This conservative approach enables enterprise pricing with minimal support overhead.
